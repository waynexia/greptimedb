async fn create_logical_region( &self, logical_region_id: RegionId, request: RegionCreateRequest ) -> Result<()>

Create a logical region.

Physical table and logical table can have multiple regions, and their region number should be the same. Thus we can infer the physical region id by simply replace the table id part in the given region id, which represent the “logical region” to request.

This method will alter the data region to add columns if necessary.

If the logical region to create already exists, this method will do nothing.

pub async fn open_region( &self, region_id: RegionId, request: RegionOpenRequest ) -> Result<AffectedRows>

Open a metric region.

Only open requests to a physical region matter. Those to logical regions are actually an empty operation – it only check if the request is valid. Since logical regions are multiplexed over physical regions, they are always “open”.

If trying to open a logical region whose physical region is not open, metric engine will throw a RegionNotFound error.
